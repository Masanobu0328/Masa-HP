# Example Blog Article Structure

This template shows the expected structure and rhythm of a Masa-style blog article.

---

## タイトル例
**人文系こそAIコーディングを学べ：言語感覚が最強の武器になる時代**

---

## 導入部

AIと聞くと、多くの人は「理系の世界」だと思っている。  
数学が得意で、プログラミングに詳しい人だけのものだと。

でも、本質的には逆だ。

これからの時代、AIを本当に使いこなせるのは「言葉を扱う力」を持っている人間だと私は思う。

---

## 本文構成例

### H2: AIは「言語」で動いている

ChatGPTやClaude、GitHub Copilot。  
これら全てのベースになっているのは、**自然言語処理**だ。

つまり、人間の言葉をどう理解し、どう変換し、どう出力するか。

ここに全てがかかっている。

プログラミング言語も、突き詰めれば「人間とコンピュータの間の翻訳言語」なんです。  
文法があり、構造があり、意味がある。

**言葉を扱う感覚が鋭ければ、プログラミングも自然に理解できる。**  
これは私が実際にAIツールを使い続けて実感したことだ。

### H2: 「曖昧さ」に耐えられる人が、AIを使いこなせる

人文系の人が持っている最大の強みは何か。

それは、**曖昧さや矛盾を受け入れられる思考の柔軟性**だと思う。

プログラミングの世界では、昔は「正確さ」「厳密さ」が絶対だった。  
一文字でも間違えれば、エラー。  
論理が破綻していれば、動かない。

でも、AIが登場してから状況は変わった。

今のAIコーディングツール（Cursor、Windsurf、Antigravityなど）では、  
「こういう感じで動くものが欲しい」  
と**自然な言葉で伝えれば**、AIがコードを生成してくれる。

完璧じゃなくてもいい。  
曖昧でもいい。  
対話しながら調整すればいい。

この「不完全でも進む」「対話で磨く」というプロセスは、  
まさに人文系が得意とする世界じゃないだろうか。

### H2: 具体例：私の場合

私は理学療法士で、プログラミングは完全独学だ。

最初は「変数って何？」「関数って何？」というレベルだった。  
でも、AIと対話しながら、毎日コードを書き続けた。

休日は1日中AIツールを触り、  
わからないところは聞き、  
間違えたら直してもらう。

半年後、私は自分の事業のためのWebアプリや歩行解析AIを作っていた。

**知識ゼロから始めても、「問いを立てる力」と「対話する力」があれば、形にできる。**

これがAI時代のコーディングなんです。

### H2: 何から始めればいいのか

もしあなたが「自分も試してみたい」と思ったなら、  
以下のステップをおすすめする。

1. **CursorかWindsurfをインストールする**  
   無料プランで十分。まずは触ってみること。

2. **簡単なものを作ってみる**  
   ブログなら1記事目を書く。  
   アプリなら「自己紹介ページ」を作る。  
   とにかく小さく始める。

3. **対話しながら改善する**  
   AIに「ここをこう変えたい」と伝え続ける。  
   エラーが出ても、AIに聞けば解決策を出してくれる。

4. **毎日触り続ける**  
   週末だけ、ではなく、毎日少しでもいいから触る。  
   これだけで、半年後には全く違う景色が見える。

---

## 結論部

AIは「理系のもの」ではない。

むしろ、**言語を扱う力、曖昧さに耐える力、対話を楽しむ力**を持った人にこそ、  
最大の武器になる。

知識はAIが持っている。  
あなたが持つべきは、**問いを立てる力**と、**それを形にする意志**だ。

それがあれば、人文系でも、文系でも、誰でも、AIと共にものを作れる時代になった。

あとは、草むらに入るだけだ。

## このテンプレートのポイント

1. **リズムの作り方**:
   - 短い文と長い文を織り交ぜる
   - 体言止めで切る（「それは、〜だ。」）
   - 問いかけを単独行で強調

2. **強調の手法**:
   - **太字 + 断定形**で重要メッセージを示す
   - 「〜なんです」で柔らかく説明
   - 「〜だ」で締めて印象づける

3. **構造**:
   - 導入で問題提起 → 本質の問い → 具体例 → アクション → 結論
   - 各セクションで「主張 → 理由 → 例 → 結論」のサイクル

4. **Masaらしさ**:
   - 自己開示（「私は〜だった」）
   - メタ認知（「〜と実感した」）
   - 実践重視（「まず触ってみること」）
   - 本質への問い（「本質的には逆だ」）
